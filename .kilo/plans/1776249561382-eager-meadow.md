# Google Ads Analytics Backend Implementation Plan

## Current State Analysis
- Backend has Google Ads API integration with automatic sync every 6 hours
- Data models: Campaign, Metric storing impressions, clicks, cost, conversions, click breakdown
- Sync service fetches and stores last 30 days of data
- Client model has google_ads_customer_id and google_ads_enabled fields
- **Missing**: No systematic way to associate Google Ads customer IDs with CRM clients
- No bulk operations for client-Google Ads mapping
- No validation or verification of Google Ads customer IDs

## Implementation Goals
- Establish robust client-Google Ads association system
- Enable automatic fetching of analytics data (spend, clicks, impressions, conversions, CTR, CPC, etc.) once associated
- Build comprehensive analytics data aggregation and querying from stored metrics
- **Create 2 key analytics endpoints**:
  1. List all clients with overview data (Call clicks, Website clicks, Fund, CPL)
  2. Single client detailed reports with date filtering capabilities
- Create automated sync and data validation processes
- Implement performance metrics calculation engine
- Set up data export and reporting infrastructure

## Phase 1: Client-Google Ads Association Foundation
### 1.1 Enhanced Client Model
- Add validation for google_ads_customer_id format (10-digit number)
- Add google_ads_account_name field for better identification
- Implement unique constraints and indexing for Google Ads fields

### 1.2 Association Management API
- `PUT /api/google-ads/client/:clientId/associate` - Associate Google Ads customer ID with client
- `POST /api/google-ads/clients/bulk-associate` - Bulk associate multiple clients
- `GET /api/google-ads/clients/unassociated` - Get clients without Google Ads IDs
- `POST /api/google-ads/validate-customer-id` - Validate Google Ads customer ID exists

### 1.3 Data Migration Utilities
- Script to bulk import client-Google Ads mappings from CSV/Excel
- Data validation and cleanup utilities
- Audit trail for association changes

## Phase 4: Analytics API Layer
### 4.1 Core Analytics Endpoints
- `GET /api/analytics/clients` - List all clients with overview data (Call clicks, Website clicks, Fund/Budget, CPL)
- `GET /api/analytics/client/:clientId` - Detailed reports for single client with date filters
- `GET /api/analytics/overview` - System-wide KPIs and summaries
- `GET /api/analytics/campaign/:campaignId` - Campaign-level metrics
- `GET /api/analytics/metrics` - Raw metrics with filtering and aggregation

### 4.2 Client Overview Data Structure
Each client overview should include:
- Total Call clicks (from click_breakdown.call_clicks)
- Total Website clicks (from click_breakdown.website_clicks)
- Fund/Budget information (from billing and campaign budgets)
- CPL (Cost Per Lead) calculations
- Basic metrics: impressions, clicks, cost, conversions

### 4.3 Detailed Client Analytics
For single client endpoint `GET /api/analytics/client/:clientId`:
- Date range filtering (start_date, end_date query parameters)
- Daily breakdown of all metrics
- Campaign-wise performance
- Click type analysis (website, call, other)
- Trend calculations (vs previous period)
- Export capabilities (CSV, JSON)

### 4.3 Reporting Infrastructure
- Scheduled report generation
- Custom report templates
- Automated data exports
- Integration hooks for external systems

## Technical Considerations
- **Data Volume**: Handle large amounts of historical data efficiently with proper indexing
- **API Rate Limits**: Respect Google Ads API quotas and implement backoff strategies
- **Currency Handling**: Consistent INR conversion and micro-to-standard unit conversions
- **Time Zones**: Proper handling of date ranges and time zone conversions
- **Error Handling**: Robust error handling for API failures and data inconsistencies
- **Data Integrity**: Ensure referential integrity between clients, campaigns, and metrics
